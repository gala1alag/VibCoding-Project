# 开发者工作台 — 需求文档

**版本：** v1.3  
**日期：** 2026-04-29  
**技术栈：** Python 3.9 + PyQt5 + SQLite + rapidfuzz + scapy/pyshark + matplotlib + Pillow + psutil

---

## 一、项目概述

基于 PyQt5 开发的 Windows 桌面工具，整合代码项目管理、学习笔记、快速启动与网络抓包等功能模块，数据本地 SQLite 存储，支持离线使用。将代码项目管理、学习笔记沉淀、快速启动、网络抓包分析合并到一个体验一致的客户端中，以模块化本地架构支撑知识标签体系、TODO 协作与学习统计能力，满足个人开发者日常开发与系统化学习的场景。

---

## 二、核心功能列表

1. 代码项目与单文件统一管理，自动识别编程语言
2. 学习笔记与富文本备注，按项目独立存储，支持 Markdown 渲染与导出
3. 代码片段库，按语言分类，语法高亮，一键复制
4. TODO 看板，三列拖拽，支持跨项目任务汇总与今日视图
5. 学习计时器，番茄钟模式，记录每个项目学习时长并可视化统计（热力图 + 趋势图）
6. 知识标签系统，跨项目聚合笔记，构建个人知识网络
7. 快速启动器，模糊匹配本机程序，支持全局快捷键呼出
8. 网络抓包，支持全局抓包与指定域名流量监控，可导出 `.pcap`
9. LeetCode Hot 100 刷题模块，本地生成 C++ 题目文件，一键编译运行

## 三、功能模块详细说明

### 模块 1：项目管理

**目标：** 统一管理本地代码项目和单文件学习代码。

**条目类型：**
- **文件夹项目**：关联一个本地文件夹路径（如一个完整的工程目录）
- **单文件代码**：关联单个源代码文件（如一个 `.cpp` 学习文件，一个文件即一个学习单元）

**自动语言识别：**
- 文件夹项目：扫描目录内文件扩展名，统计占比，自动打上语言标签
- 单文件：直接根据扩展名识别
- 支持语言：C++、Python、Java、JavaScript、TypeScript、C#、Go、Rust、HTML/CSS、其他

**每个条目包含：**
- 名称
- 路径（文件夹或文件）
- 语言标签（自动识别，可手动修改）
- 自定义颜色标签（用于视觉区分）
- 备注（单行短文本）
- 学习笔记（Markdown 富文本）
- 默认编辑器（可单独设置，支持自定义路径）
- 创建时间 / 最后访问时间

**操作：**
- 添加、编辑、删除条目
- 一键用指定编辑器打开（PyCharm / Cursor / Qt Creator / Visual Studio 及自定义编辑器）
- 可为每个条目设置默认编辑器，也可临时选择其他编辑器打开
- 项目活跃度排行（按最近打开时间或累计学习时长排序）

### 模块 2：学习笔记

**目标：** 为每个项目/文件提供独立的笔记空间，记录学习过程。

**功能：**
- 每个条目拥有独立笔记页
- 支持 Markdown 语法，编辑区与预览区左右分栏
- 笔记内容持久化存储到 SQLite
- 一键导出为 PDF 或 HTML 文件
- 在项目详情面板内直接编辑，无需跳转

---

### 模块 3：快速启动器

**目标：** 通过自然语言输入快速启动电脑中的任意程序，无需调用 AI 模型。

**实现方式：**
- 启动时扫描以下路径，建立程序索引：
  - `C:/ProgramData/Microsoft/Windows/Start Menu`
  - `C:/Users/{用户名}/AppData/Roaming/Microsoft/Windows/Start Menu`
  - 桌面（用户桌面 + 公共桌面）
  - `C:/Program Files` / `C:/Program Files (x86)`（仅一级目录）
- 收集所有 `.exe` 和 `.lnk` 快捷方式，提取程序名
- 用户在搜索框输入关键词（中英文均可），使用 `rapidfuzz` 模糊匹配，返回 Top 10 结果
- 显示匹配结果列表，点击或回车启动
- 支持全局快捷键（`Alt + Space`）呼出，无需打开主窗口
- 支持手动刷新索引

**示例：**
- 输入 "pycharm" → 匹配并启动 PyCharm
- 输入 "微信" → 匹配并启动微信
- 输入 "cursor" → 匹配并启动 Cursor

### 模块 4：全局搜索

**目标：** 跨模块快速定位内容。

**搜索范围：**
- 项目名称、备注、笔记内容
- 代码片段标题和内容
- 知识标签
- 语言标签

**功能：**
- 顶部搜索栏，实时过滤结果
- 按语言标签筛选（下拉或标签点击）
- 搜索结果高亮显示匹配关键词

---

### 模块 5：网络抓包

**目标：** 实时捕获和展示本机网络数据包，支持全局抓包和指定目标抓包。

**依赖：** scapy + pyshark（需 Npcap 驱动，已确认安装）

**全局抓包模式：**
- 列出本机所有可用网卡，用户选择后开始抓包
- 实时显示数据包列表，列：序号、时间、协议、源IP、源端口、目标IP、目标端口、数据长度
- 开始 / 停止 按钮控制抓包
- 按协议过滤：TCP / UDP / HTTP / ICMP / DNS / 全部
- 点击单条数据包，右侧显示原始包详情
- 支持导出为 `.pcap` 文件，可用 Wireshark 二次分析

**指定目标模式：**
- 用户输入域名或 IP（如 `github.com`、`8.8.8.8`）
- 自动解析域名为 IP
- 只捕获与该 IP 相关的数据包（源或目标匹配）
- 动态展示数据包往返情况（发送/接收方向标识）

**注意：** 抓包功能需要以管理员权限运行程序，启动时自动检测并提示。

---

### 模块 6：代码片段库

**目标：** 保存和复用常用代码片段，按语言分类管理。

**功能：**
- 新建片段：标题、语言、代码内容、描述
- 按语言分类浏览（与项目管理共用语言标签体系）
- 代码内容支持语法高亮显示
- 一键复制片段到剪贴板
- 支持搜索片段标题和内容
- 片段可关联到某个项目（可选）

---

### 模块 7：TODO 看板

**目标：** 为每个项目提供独立的任务管理看板。

**功能：**
- 每个项目拥有独立看板
- 三列状态：待办 / 进行中 / 已完成
- 任务卡片包含：标题、描述（可选）、优先级（高/中/低）、截止日期（可选）
- 支持拖拽卡片在列间移动
- 今日视图：只显示截止日期为今天或已逾期的任务
- 汇总视图：所有项目未完成任务，按项目分组展示

---

### 模块 8：学习计时器

**目标：** 记录每个项目的学习时长，帮助量化学习投入。

**功能：**
- 番茄钟模式：25 分钟专注 + 5 分钟休息，可自定义时长
- 在项目详情页一键启动计时，自动关联当前项目
- 计时结束后自动记录一条学习记录（项目、时长、日期）
- 专注模式：进入后只显示当前项目 TODO 和计时器，隐藏其他模块
- 统计面板：
  - 学习热力图（类似 GitHub contribution graph）
  - 最近 7 天 / 30 天学习时长趋势图
  - 各语言学习时长占比饼图
  - 今日学习总时长（顶部导航栏常驻显示）
  - 周报摘要：本周时长、完成 TODO 数、新增笔记数、最活跃项目

---

### 模块 9：知识标签系统

**目标：** 跨项目聚合相关知识点，建立个人知识网络。

**功能：**
- 给笔记内容打自定义标签（如 #算法、#指针、#设计模式、#网络）
- 标签管理页：列出所有标签及其关联的笔记数量
- 点击标签，聚合显示所有带该标签的笔记片段（跨项目）
- 标签支持颜色自定义

---

### 模块 11：LeetCode Hot 100 刷题

**目标：** 本地管理 LeetCode Hot 100 题目，生成 C++ 文件，直接在工作台内编译运行。

**题目数据：**
- 内置完整 Hot 100 题目列表（100 条），硬编码为本地 Python 字典
- 每条数据包含：题号、英文标题、中文标题、难度（Easy/Medium/Hard）、标签（数组、链表、动态规划等）
- 无需联网，完全离线可用

**功能：**
- 题目列表视图：按难度/标签筛选，显示完成状态（未开始/进行中/已完成）
- 一键生成 `.cpp` 文件：按题号命名（如 `0001_two_sum.cpp`），文件头自动生成注释模板：
  ```cpp
  /*
   * LeetCode #1 - 两数之和 (Two Sum)
   * 难度: Easy
   * 标签: 数组, 哈希表
   */
  #include <bits/stdc++.h>
  using namespace std;

  // 在此编写你的解法
  class Solution {
  public:

  };
  ```
- 生成后自动用指定编辑器打开（默认 Cursor 或 VS Code）
- 一键编译：调用 `g++` 编译当前题目文件，终端内显示编译输出
- 一键运行：编译成功后直接运行，输出结果显示在内嵌终端区域
- 每道题支持添加学习笔记和标记完成状态
- 统计面板：已完成题数、各难度完成比例、连续刷题天数

**文件存放路径：** 用户可自定义，默认 `~/LeetCode/`

**依赖：** 需要系统安装 `g++`（MinGW 或 MSVC）

**数据存储：** `leetcode_progress` 表：id, problem_id, status, note, file_path, created_at, completed_at

**功能：**

## 四、界面设计

**整体布局：**
```
┌──────────────────────────────────────────────────────────────────┐
│  顶部导航栏：模块切换 + 全局搜索框 + 今日学习时长                   │
├──────────────┬───────────────────────────────────────────────────┤
│              │                                                   │
│  左侧列表    │         右侧详情 / 操作面板                         │
│  （项目列表  │  （笔记、TODO看板、计时器、片段、标签等）             │
│   或程序列表 │                                                   │
│   或数据包列表）                                                  │
│              │                                                   │
└──────────────┴───────────────────────────────────────────────────┘
```

**导航模块：** 今日概览 / 项目管理 / 代码片段库 / TODO 看板 / 学习统计 / 快速启动 / 网络抓包 / 知识标签

**主题：** 深色主题（默认），支持浅色主题切换
- 深色：背景 `#1E1E1E`，强调色 `#4A90D9`，文字 `#D4D4D4`
- QSS 样式表全局应用，主题文件分开存放

**编辑器图标：** 在打开按钮旁显示对应编辑器图标（使用 QtAwesome）

## 八、已知约束

- 网络抓包需要管理员权限运行，启动时自动检测
- 快速启动器首次扫描可能需要数秒，需加进度提示
- Markdown 渲染依赖 PyQt5 5.14+（当前 5.15.9，满足要求）
- matplotlib 嵌入 PyQt5 需使用 `FigureCanvasQTAgg`
- 数据库字段变更需处理旧版本兼容升级
- 不支持 macOS / Linux（Windows 专属）
