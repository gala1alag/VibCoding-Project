# 开发者工作台 — 头脑风暴文档

**版本：** v1.1  
**日期：** 2026-04-29

---

## 一、产品定位

一款专为 Windows 开发者设计的本地桌面工具，将代码项目管理、学习笔记、快速启动、网络抓包整合进同一个界面，让开发者不再在多个工具之间来回切换。所有数据本地存储，无需联网，支持离线使用。

---

## 二、核心功能

1. 代码项目与单文件统一管理，自动识别编程语言
2. 学习笔记与富文本备注，按项目独立存储
3. 代码片段库，按语言分类，一键复制
4. TODO 看板，三列拖拽，支持跨项目任务汇总
5. 学习计时器，番茄钟模式，记录每个项目学习时长并可视化统计
6. 知识标签系统，跨项目聚合笔记，构建个人知识网络
7. 快速启动器，模糊匹配本机程序，自然语言输入直接启动
8. 网络抓包，支持全局抓包与指定域名流量监控

---

## 三、体验层面的想法

### 今日概览面板
启动时展示一个仪表盘：今天学了多久、有几个待办任务、最近打开的项目列表。让用户每次打开工作台都能快速进入状态，而不是面对一个空白列表。

### 项目卡片个性化
每个项目支持自定义颜色标签或封面图，视觉上一眼区分不同项目。对于同时维护多个项目的开发者，颜色比文字更快速。

### 全局快捷键呼出快速启动器
快速启动器支持全局快捷键（如 `Alt + Space`）呼出，不需要打开主窗口就能搜索并启动程序。类似 macOS Spotlight 的体验，减少切换成本。

### 专注模式
进入专注模式后，界面只保留当前项目的 TODO 列表和计时器，隐藏其他所有模块，减少干扰。

## 四、功能延伸的想法

### Markdown 笔记渲染
学习笔记支持 Markdown 语法，编辑区和预览区左右分栏显示。写完后可以一键导出为 PDF 或 HTML 文件，方便分享或归档。

### 代码片段语法高亮
代码片段库中的代码内容支持语法高亮显示，根据语言标签自动应用对应的高亮规则，阅读体验更接近编辑器。

### TODO 今日视图
TODO 看板新增"今日"视图，只显示截止日期为今天或已逾期的任务，配合学习计时器使用，聚焦当天目标。

### 抓包结果导出
网络抓包结果支持导出为 `.pcap` 文件，可以直接用 Wireshark 打开进行二次深度分析，工作台作为轻量入口，Wireshark 作为专业后端。

### 编辑器路径自定义
编辑器不局限于预设的四个（PyCharm、Cursor、Qt Creator、Visual Studio），支持用户手动添加任意编辑器的可执行文件路径，灵活适配不同工作流。

## 五、数据与统计的想法

### 学习热力图
类似 GitHub contribution graph，以日历热力图的形式展示每天的学习时长，颜色深浅代表投入程度。直观看出学习连续性和活跃周期。

### 语言学习占比
饼图展示各编程语言的累计学习时长占比，帮助用户了解自己的技术方向分布。

### 项目活跃度排行
按最近打开时间或累计学习时长对项目排序，快速找到最近在推进的项目，也能发现长期搁置的项目。

### 周报摘要
每周自动生成一份学习摘要：本周学习总时长、完成的 TODO 数量、新增笔记数、最活跃的项目。可以复制文本或导出。

---

## 六、个性化的想法

### 浅色 / 深色主题切换
支持手动切换浅色和深色主题，同时跟随系统主题自动切换（可选）。

### 数据备份与恢复
支持将所有数据（项目列表、笔记、片段、TODO、学习记录）导出为单个 JSON 文件，也可以从备份文件恢复，防止数据丢失或迁移到新电脑。

### 启动项配置
支持设置开机自启动，以及启动时默认打开的模块（如默认打开今日概览或上次使用的模块）。

## 七、技术栈

| 层级 | 技术 |
|------|------|
| 界面框架 | PyQt5 |
| 图标库 | QtAwesome |
| 图表 | matplotlib |
| 数据存储 | SQLite（内置 sqlite3） |
| 模糊匹配 | rapidfuzz |
| 网络抓包 | scapy + pyshark |
| 运行环境 | Python 3.9，Windows 10/11 |
| 抓包驱动 | Npcap（已安装） |

---

## 八、潜在风险与约束

| 风险 | 说明 |
|------|------|
| 抓包需管理员权限 | 需要以管理员身份运行程序，否则 scapy 无法捕获数据包 |
| 快速启动器首次扫描慢 | Program Files 目录较大，首次建立索引可能需要数秒，需加进度提示 |
| matplotlib 嵌入 PyQt5 | 需要使用 FigureCanvasQTAgg，有一定集成复杂度 |
| 数据库迁移 | 后续新增字段需要处理旧版本数据库的兼容升级 |
| Windows 专属 | 快速启动器和抓包模块依赖 Windows API，不支持跨平台 |

---

## 九、开发优先级建议

| 阶段 | 模块 | 理由 |
|------|------|------|
| 第一阶段 | 项目管理 + 快速启动器 | 核心骨架，验证整体框架可行性 |
| 第二阶段 | 学习笔记 + 代码片段库 + TODO 看板 | 内容沉淀类功能，依赖第一阶段数据结构 |
| 第三阶段 | 学习计时器 + 知识标签 + 统计图表 | 数据积累后才有意义的功能 |
| 第四阶段 | 网络抓包 + 今日概览 + 数据备份 | 独立模块，可单独开发集成 |
